Part A: Find Mistakes (Identify + Fix)

For each item: (1) identify the issue in one sentence, (2) write the corrected HTML.

Common HTML structure and accessibility mistakes (practice format)
Issue Wrong HTML Correct HTML
Navigation link inside ul without li 
<nav>
  <ul>
    <a href="index.html">Home</a>
    <li><a href="models.html">Models</a></li>
  </ul>
</nav>
 
<nav aria-label="Primary navigation">
  <ul>
    <li><a href="index.html">Home</a></li>
    <li><a href="models.html">Models</a></li>
  </ul>
</nav>
More than one aria-current="page" in the same menu 
<nav aria-label="Primary navigation">
  <ul>
    <li><a href="index.html" aria-current="page">Home</a></li>
    <li><a href="practice.html" aria-current="page">Practice</a></li>
  </ul>
</nav>
 
<nav aria-label="Primary navigation">
  <ul>
    <li><a href="index.html">Home</a></li>
    <li><a href="practice.html" aria-current="page">Practice</a></li>
  </ul>
</nav>
Label not connected to control (for/id mismatch) 
<label for="msg">Message</label>
<textarea id="message" name="message"></textarea>
 
<label for="message">Message</label>
<textarea id="message" name="message"></textarea>
Heading hierarchy skips a level 
<h1>Interaction Analysis</h1>
<h3>Norman vs GOMS</h3>
 
<h1>Interaction Analysis</h1>
<h2>Norman vs GOMS</h2>
Comparison table missing caption + header cells 
<table>
  <tr><td>Model</td><td>Predicts</td></tr>
  <tr><td>KLM</td><td>Time</td></tr>
</table>
 
<table>
  <caption>Models and what they predict</caption>
  <thead>
    <tr>
      <th scope="col">Model</th>
      <th scope="col">Predicts</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th scope="row">KLM</th>
      <td>Execution time</td>
    </tr>
  </tbody>
</table>

Part B: True / False

  • T/F: Every link inside <ul> must be inside an <li>.
  • T/F: A page may have multiple <h1> headings to make text larger.
  • T/F: aria-current="page" should be used only on the active navigation link.
  • T/F: A form is accessible even if it has no <label> elements.
  • T/F: KLM predicts execution time for expert, error-free performance.

Part C: Short Answer

  • Explain the difference between a semantic element (e.g., nav) and a generic element (e.g., div).
  • Why does KLM not model high-level planning and interpretation?

Hint: KLM is an execution-level model. It predicts performance time using low-level operators, and does not model interpretation, learning, or feedback processing.
