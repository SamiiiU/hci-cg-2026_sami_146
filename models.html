Interaction Models in HCI

This page explains three core interaction models used in Human–Computer Interaction: Norman’s Action Cycle, GOMS, and the Keystroke-Level Model (KLM).

Norman’s Action Cycle (Chapter 3)

Norman’s Action Cycle describes interaction as a sequence of cognitive stages that occur when a user attempts to achieve a goal using a system.

  1. Form goal
  2. Form intention
  3. Specify action
  4. Execute action
  5. Perceive system state
  6. Interpret system state
  7. Evaluate outcome

The model highlights two key problems: the Gulf of Execution (difficulty of performing actions) and the Gulf of Evaluation (difficulty of understanding feedback).

Purpose: Norman’s model is descriptive. It explains why users make mistakes and how interface feedback affects understanding.

GOMS Model (Chapter 12, Page# 422-424)

GOMS stands for Goals, Operators, Methods, and Selection Rules. It models how an expert user performs a task in a structured, predictable manner.

  • Goals: What the user wants to achieve
  • Operators: Basic actions such as clicking or typing
  • Methods: Sequences of operators to achieve a goal
  • Selection Rules: Rules for choosing between methods

GOMS assumes error-free performance and does not model learning, confusion, or interpretation.

Purpose: GOMS is an analytic model used to compare task structures and efficiency.

Keystroke-Level Model (KLM) (Chapter 12 , Page# 436-437)

The Keystroke-Level Model (KLM) is a low-level instance of GOMS. It predicts how long an expert user will take to complete a task.

KLM decomposes interaction into a small set of primitive operators:

  • K – Keystroke or button press
  • P – Pointing to a target
  • H – Homing (switching hands)
  • M – Mental preparation
  • R – System response

Each operator has an associated time value, allowing prediction of execution time for expert, error-free interaction.

Purpose: KLM focuses on execution speed only. It does not model planning, interpretation, or feedback.

Comparison Summary (Dix et al., Ch. 12, pp. 441–442)

Comparison of Norman, GOMS, and KLM
Model Type Explains Predicts
Norman Descriptive Understanding, feedback, errors Nothing quantitative
GOMS Analytic Task structure for experts Relative efficiency
KLM Predictive Execution-level actions Execution time
